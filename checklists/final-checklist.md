# Final Technical CV Checklist

Use this checklist before every application. A checked box should mean you verified it, not that you assume it is correct.

## Target and relevance

- [ ] The target role is clear within the first third of the page.
- [ ] The CV is tailored to one role or a closely related role family.
- [ ] The most relevant evidence appears before weaker evidence.
- [ ] Irrelevant details were removed.
- [ ] The summary matches the rest of the CV.
- [ ] Important job requirements are supported by truthful evidence.

## Contact and links

- [ ] Name is written consistently.
- [ ] Email address is professional.
- [ ] Phone number includes the correct country code when needed.
- [ ] City and country are accurate.
- [ ] LinkedIn link opens correctly.
- [ ] GitHub and portfolio links open correctly.
- [ ] Linked repositories are documented and presentable.
- [ ] The public version does not expose unnecessary personal data.

## Technical skills

- [ ] Skills are grouped by category.
- [ ] Every listed tool can be explained in an interview.
- [ ] Capitalization and product names are correct.
- [ ] There are no bars, stars, or unsupported percentages.
- [ ] Core skills appear in project or experience evidence.
- [ ] Basic exposure is not presented as expertise.

## Projects and experience

- [ ] Each entry makes your contribution clear.
- [ ] Bullets start with direct action verbs.
- [ ] Bullets describe how technologies were used.
- [ ] Team achievements are not presented as individual achievements.
- [ ] Metrics are real and reproducible.
- [ ] Projects include testing, validation, deployment, or purpose where relevant.
- [ ] Verb tense is consistent.
- [ ] Current and past dates are accurate.

## Format and readability

- [ ] Section headings are conventional and easy to scan.
- [ ] Font size is readable at normal zoom.
- [ ] Spacing and margins are consistent.
- [ ] Date format is consistent.
- [ ] Bullets and punctuation are consistent.
- [ ] The layout is not crowded.
- [ ] No important content depends only on an icon or color.
- [ ] The CV remains clear when printed in grayscale.

## ATS and file checks

- [ ] The employer's required file format was followed.
- [ ] Text can be selected in the exported PDF.
- [ ] Copying all PDF text into a plain-text editor preserves a logical order.
- [ ] No content disappears during export.
- [ ] Hyperlinks work in the exported file.
- [ ] The file opens on another device or PDF reader.
- [ ] The filename is clear, for example `Name_Role_CV.pdf`.
- [ ] Document metadata and comments do not contain unwanted template information.

## Language and credibility

- [ ] Spelling and grammar were checked.
- [ ] Another person or tool reviewed the document.
- [ ] Unsupported adjectives were removed.
- [ ] There are no fake or AI-invented claims.
- [ ] Acronyms are understandable in context.
- [ ] Every claim can be defended with an explanation or evidence.
- [ ] Confidential information, credentials, and secrets are absent.

## Application record

- [ ] A copy of this exact CV version was saved.
- [ ] The job description or URL was saved.
- [ ] The application date was recorded.
- [ ] The company and role name were recorded.
- [ ] Follow-up or interview notes have a place to be stored.

If several boxes remain unchecked, fix the document before submitting. Sending more applications with a weak generic CV is not the same as improving the search.
