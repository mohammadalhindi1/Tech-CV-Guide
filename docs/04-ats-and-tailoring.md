# ATS and Honest CV Tailoring

## What an ATS is

An applicant tracking system helps employers collect, organize, search, and manage applications. Different employers configure different systems and hiring workflows. There is no universal ATS score and no guaranteed trick that moves a CV to an interview.

Your practical goal is simpler:

1. make important text extractable;
2. use clear headings;
3. use relevant terminology honestly;
4. make the document useful to the human who reads it.

## Safe formatting principles

Prefer:

- standard section headings;
- selectable text;
- common fonts;
- consistent dates;
- simple bullets;
- clear reading order;
- ordinary hyperlinks with visible labels;
- PDF or DOCX according to the employer's instructions.

Use caution with:

- multi-column layouts;
- text boxes;
- icons used instead of words;
- important information in headers or footers;
- charts, logos, and skill bars;
- scanned or image-only PDFs;
- unusual section names;
- tables that produce a broken reading order.

This does not mean every ATS rejects every column or icon. It means complex design creates avoidable parsing and accessibility risk.

## The copy-and-paste test

After exporting the CV:

1. open the PDF;
2. select all text;
3. paste it into a plain-text editor;
4. confirm that name, headings, dates, titles, bullets, and links appear in a logical order.

This is not a complete ATS simulation, but it catches many extraction problems.

## Tailoring workflow

### Step 1: Read for responsibilities

Identify the three to five recurring outcomes. For example:

- maintain cloud infrastructure;
- automate CI/CD;
- troubleshoot Linux systems;
- deploy containers;
- collaborate with developers.

### Step 2: Read for evidence requirements

Extract required technologies, environment, seniority, and domain knowledge.

### Step 3: Classify each requirement

| Class | Meaning | Action |
|---|---|---|
| Proven | You have credible evidence | Include it prominently |
| Adjacent | You used a related tool or concept | Describe it accurately |
| Learning | You are actively building it | Include only with clear context |
| Missing | You have no evidence | Do not claim it |

### Step 4: Adjust the CV

Tailoring may change:

- professional summary;
- skill order;
- project order;
- selected bullets;
- exact truthful terminology;
- filename.

Tailoring should not change facts, dates, job titles, or the level of your experience.

## Keyword use

If a job says “CI/CD pipelines” and you built workflows with GitHub Actions, use both the concept and the tool naturally:

> Built CI/CD checks with GitHub Actions to compile and test the project on each push.

Do not paste a hidden list of keywords, repeat terms unnaturally, or claim every item in the job description.

## PDF or DOCX?

Follow the application instruction first.

- PDF usually preserves layout.
- DOCX may be explicitly requested by recruiters or application systems.
- Either format can fail when the underlying document is poorly structured.
- Keep an editable source file and test the exported version.

## ATS myths to reject

- “Every CV must contain exactly the same keywords as the job post.”
- “White text with hidden keywords improves ranking.”
- “A purchased ATS score predicts whether you will be interviewed.”
- “One template works identically in every hiring system.”
- “The ATS makes the final hiring decision by itself.”

## Human review still matters

Even a perfectly extracted CV can fail when it is generic, exaggerated, difficult to scan, or unsupported. Technical hiring teams often inspect projects, GitHub repositories, depth of knowledge, and the candidate's ability to explain decisions.

Optimize for **clarity, relevance, and credibility**, not for a fictional robot.
